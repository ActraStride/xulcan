# Xulcan

可用语言：[英语](../README.md) · [西班牙语](README.es.md) · [法语](README.fr.md) · [俄语](README.ru.md) · [阿拉伯语](README.ar.md) · [中文（普通话）](README.zh.md)

**Xulcan 是一个遵循 “API 优先（API-first）” 理念的后端框架，用于构建、运行和部署先进的人工智能智能体（AI Agents）。**

其使命是抽象大模型编排、记忆管理以及工具调用的复杂性，使开发者能够通过声明式配置与健壮的 REST API，将复杂的推理能力无缝集成到各类应用程序中。

项目以开源方式发布，但目前以个人平台的形式精心打造，目标是成长为完整、完善的智能体开发生态系统。

---

## 愿景与理念

Xulcan 的灵感来自 **带有通用适配器的建筑师圆规**。这一意象体现了我们的核心设计哲学：

* **精准：** 将含糊的自然语言转化为结构化且准确的行动。
* **模块化：** 通过可插拔组件（如工具、记忆）构建智能体。
* **编排：** 通过有序的推理过程，将上下文（固定点）与行动（移动点）连接起来。

我们坚持 **“配置即代码（Configuration as Code）”** 与 **“API 优先”** 的设计原则，以确保系统的解耦性、可维护性和可扩展性。

---

## 概念架构

Xulcan 被设计为一个托管式服务，通过 API 与客户端应用进行交互。

1. **Xulcan 仪表盘：** 通过 Web 界面定义智能体、工具与记忆。
2. **智能体核心（引擎）：**

   * **`LLMClient`：** 面向 Gemini、OpenAI、Anthropic 等提供商的通用适配器。
   * **`ToolExecutor`：** 工具的安全执行层。
   * **`MemoryManager`：** 包含短期记忆（Redis）与长期记忆（Faiss）。
   * **`Executor`：** 推理与行动编排（Chain of Thought）。
3. **客户端集成：** 通过 REST API 提供安全、统一、简洁的访问方式。

## 🛠 开发与贡献流程

为保持系统稳定并有效管理版本发布，本项目遵循 **Git Flow** 工作流。

### 分支策略

* **`main`：** 🔴 **生产环境。** 仅保留稳定、已版本化且可直接部署的代码；禁止直接提交。
* **`develop`：** 🟡 **集成（下一个发布版本）。** 核心开发分支，所有新功能都会先合并到此处进行整体验证。
* **`feature/*`：** 🟢 **功能开发分支。** 用于开发新特性（如 `feature/infra-logging`）。

  * 分支自：`develop`
  * 合并回：`develop`
* **`hotfix/*`：** 🚑 **紧急修复。** 针对生产环境中的关键问题，从 `main` 分出并合并回 `main` 和 `develop`。

### 提交规范

遵循 [Conventional Commits](https://www.conventionalcommits.org/)，以保持语义化的提交历史：

* `feat:` 新增功能
* `fix:` 修复缺陷
* `chore:` 维护或配置更新
* `refactor:` 不影响行为的代码重构

### Pull Request 与合并策略

1. **Feature → Develop：**

   * 使用 **Squash and Merge**
   * *目标：* 使每个特性在 `develop` 历史中呈现为单一、干净的提交。

2. **Develop → Main（发布）：**

   * 使用 **Merge Commit**（创建合并提交）
   * *目标：* 明确保留一个版本（如 v0.1.0）中包含的功能集合。

3. **测试验证：**
   所有 PR 必须通过 CI（Docker 构建 + Pytest）方可合并。
